sys_cache_info: La memorizzazione nella cache è essenziale per velocizzare il tuo sito e dovrebbe essere generalmente abilitata. Tuttavia, in alcune circostanze potresti voler disabilitare o svuotare la cache.

-

sys_cache_clear: Cancella cache

-

sys_cache_clear_info: La cancellazione della cache del sito impone una nuova scansione delle pagine e dei file aggiornati. Questa azione non rimuoverà alcun file di pagina o immagine dalla directory della cache.

-

sys_cache_monitor: L'intervallo di controllo è

-

sys_cache_monitor_info: Quando la memorizzazione nella cache è abilitata, Automad controllerà regolarmente le modifiche nel file system. La frequenza di questi controlli può essere definita qui.

-

sys_cache_lifetime: La durata della cache è

-

sys_cache_lifetime_info: Tutti i dati e le pagine di Automad memorizzati nella cache hanno una durata massima. Dopo che la durata di un oggetto è stata superata, la sua cache verrà ricostruita quando richiesto.

-

sys_cache_purge: Elimina directory cache

-

sys_cache_purge_info: In contrasto con la semplice cancellazione della cache, l'eliminazione della directory della cache sposterà anche tutti i file temporanei, le pagine eliminate, i backup di aggiornamento e le immagini nella cache nella directory temporanea del sistema. Ciò forzerà una ricostruzione completa dell'immagine e della cache della pagina, nonché una pulizia di tutte le pagine eliminate. Sulla maggior parte dei sistemi la directory temporanea verrà eliminata automaticamente di tanto in tanto o al riavvio. Tuttavia, su alcuni sistemi tale directory deve essere cancellata manualmente.

-

sys_debug_info: Quando il debug è abilitato, tutti i processi di Automad verranno registrati nella console del browser. Il debug è necessario solo per lo sviluppo o la risoluzione dei problemi e dovrebbe essere disabilitato in tutti gli altri casi.

-

sys_headless_edit_info: La struttura dei dati JSON restituiti è controllata da un modello. È possibile modificare quel modello utilizzando il linguaggio del modello integrato di Automad per personalizzare i campi utilizzando variabili, oggetti e strutture di controllo.

-

sys_headless_info: L'attivazione della modalità senza testa trasforma Automad in un sistema di gestione dei contenuti incentrato interamente sulla creazione di contenuti senza un livello di presentazione. Sebbene il dashboard possa ancora essere utilizzato per gestire i contenuti, tutti i dati e i file sono accessibili tramite un'API JSON di sola lettura.
