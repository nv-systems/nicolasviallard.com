sys_cache_info: O cache é essencial para acelerar seu site e geralmente deve ser ativado. No entanto, em algumas circunstâncias, você pode desativar ou limpar o cache.

-

sys_cache_clear: Limpar cache

-

sys_cache_clear_info: Limpar o cache do site força uma nova varredura de páginas e arquivos atualizados. Esta ação não removerá nenhum arquivo de página ou imagem do diretório de cache.

-

sys_cache_monitor: O intervalo de verificação é

-

sys_cache_monitor_info: Quando o armazenamento em cache está ativado, o Automad verifica regularmente se há mudanças no sistema de arquivos. A frequência dessas verificações pode ser definida aqui.

-

sys_cache_lifetime: O tempo de vida do cache é

-

sys_cache_lifetime_info: Todos os dados e páginas do Automad em cache têm uma vida útil máxima. Após o tempo de vida de um objeto exceder, seu cache será reconstruído quando solicitado.

-

sys_cache_purge: Limpar diretório de cache

-

sys_cache_purge_info: Em contraste com simplesmente limpar o cache, limpar o diretório de cache também moverá todos os arquivos temporários, páginas excluídas, backups de atualização e imagens em cache para o diretório temporário do sistema. Isso forçará uma reconstrução completa do cache de imagem e página, bem como uma limpeza de todas as páginas excluídas. Na maioria dos sistemas, o diretório temporário será limpo automaticamente de tempos em tempos ou na reinicialização. No entanto, em alguns sistemas, esse diretório deve ser limpo manualmente.

-

sys_debug_info: Quando a depuração está habilitada, todos os processos do Automad serão registrados no console do seu navegador. A depuração é necessária apenas para desenvolvimento ou solução de problemas e deve ser desativada em todos os outros casos.

-

sys_headless_edit_info: A estrutura dos dados JSON retornados é controlada por um modelo. Você pode editar esse modelo usando a linguagem de modelo embutida do Automad para personalizar campos usando variáveis, objetos e estruturas de controle.

-

sys_headless_info: Ativar o modo headless transforma o Automad em um sistema de gerenciamento de conteúdo com foco inteiramente na criação de conteúdo sem uma camada de apresentação. Embora o painel ainda possa ser usado para gerenciar conteúdo, todos os dados e arquivos podem ser acessados por meio de uma API JSON somente leitura.
